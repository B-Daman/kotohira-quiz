### 問題追加の仕組み化
- [x] **URL起点の問題作成ワークフローを確立**（2026-04-05〜運用中）
  1. あっきーさんがURLを提供する
  2. Claude CodeがURLの内容を取得・分析する
  3. 既存問題に出典が不足していれば `source` を追加・補足する
  4. URLの内容に基づいて新規問題を作成する（問題数はURL内の情報量に応じて判断）
  5. 新規問題は `reviewed: false` で作成し、両ファイル（hisho-bot正典・kotohira-quiz Web用）に追加する
  6. あっきーさんがレビューし、OKなら `reviewed: true` に更新する
- [x] **ナレッジマップ起点の一括問題作成**（2026-04-06）

### 問題作成・レビューのワークフロー（確立済み）
1. ナレッジマップで不足テーマを特定
2. テーマを4チーム程度に分担し、エージェントチームで並行作成
3. 各エージェントはWebでURL調査→裏付けが取れた分だけ問題作成（IDなし、テキスト出力）
4. メインが統合・重複チェック・ID採番・JSON書き込み
5. レビュー時にWikipedia以外のソースで裏取り→追加出典を付与

### 出典（ソース）の信頼性基準
- **Tier 1（高信頼）**: 公式サイト（konpira.or.jp, town.kotohira.kagawa.jp等）、新聞社（四国新聞等）、学術資料（美術館図録等）、政府系（文化遺産オンライン、国土地理院等）
- **Tier 2（中信頼）**: Wikipedia — 誰でも編集可能なため、Tier 1の裏付けなしでは単独出典として不十分。レビュー時に必ずTier 1での追加裏取りを行う
- **Tier 3（参考）**: 個人ブログ、観光情報サイト — 補助的な参考として使用可。単独出典には不適
