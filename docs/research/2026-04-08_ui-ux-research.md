# クイズWebアプリ UI/UX調査レポート

調査日: 2026-04-08

## タスク1: 既存クイズWebアプリのUI調査

### 1. Kahoot! (kahoot.it)

**スタート画面**
- PINコード入力のみの超ミニマルUI。ホストが作成したゲームに参加する形式
- 背景はブランドカラーの紫（#46178F）で統一
- 入力フィールドは白背景・大きなフォント・中央配置

**問題表示画面**
- 選択肢は2x2グリッド配置（4択）。各選択肢に固有の色+図形アイコン（赤=三角、青=ひし形、黄=丸、緑=四角）
- 画面上部にカウントダウンタイマー（秒数表示+プログレスバー）
- 問題文は画面中央上部に白文字・大フォントで表示
- 進捗は「問題N/全体M」で表示

**正解/不正解フィードバック**
- 正解時: 緑背景+チェックマーク+効果音（軽快なジングル）+ポイント表示
- 不正解時: 赤背景+バツマーク+残念な効果音+正解の表示
- 回答速度に応じたボーナスポイント（速いほど高得点）
- ストリーク（連続正解）ボーナス表示

**結果画面**
- 表彰台形式のランキング表示（1位/2位/3位にアニメーション）
- 紙吹雪エフェクト
- スコア詳細+正答率の表示
- 「Play again」ボタンが目立つ位置

**モバイル対応**
- 完全モバイルファースト設計（参加者側は全てスマホ前提）
- 選択肢ボタンは画面幅いっぱい、タップ領域が非常に大きい（最低56px高さ）
- 縦向き固定レイアウト

**色・フォント・余白**
- ブランドカラー: 紫(#46178F)をベースに鮮やかな4色
- フォント: Montserrat（太め、読みやすい）
- 余白: 大きめ。情報密度を低く保ち、一目で判断できる設計
- コントラスト: 背景と文字の明度差が大きい

**ゲーミフィケーション**
- リアルタイムランキング、ポイントシステム、ストリークボーナス
- 音楽とサウンドエフェクト、カウントダウンの緊張感
- 対戦形式による競争心の刺激

**色・フォント・余白**

**ゲーミフィケーション**

### 7. デザインガイドライン調査

#### Material Design 3 (MD3)

**チップコンポーネント**
- Filter Chip: 選択/非選択の2状態。選択時はトーンカラー背景+チェックマーク
- 推奨高さ: 32dp
- タッチターゲット: 最低48dp（チップ自体は32dpでも、タッチ領域は48dp確保）
- チップ間のギャップ: 8dp
- 複数選択可能なフィルターチップグループ
- アイコン付きチップで視認性向上

**カードコンポーネント**
- Elevated / Filled / Outlined の3種類
- 角丸: 12dp（MD3標準）
- 内部パディング: 16dp

**ボタンコンポーネント**
- Filled Button高さ: 40dp
- タッチターゲット: 最低48dp
- パディング: 左右24dp
- ラベルテキスト: 14sp, medium weight

#### Apple Human Interface Guidelines (HIG)

**タッチターゲット**
- 最小タップ領域: 44x44pt（約44x44px @1x）
- ボタン間のスペース: 最低8pt推奨
- 重要なアクションほど大きなタップ領域を確保

**ボタンデザイン**
- 明確な視覚的アフォーダンス（ボタンに見えること）
- プライマリアクションは塗りつぶし、セカンダリはアウトライン
- 十分なコントラスト比（4.5:1以上）

## タスク2: 分析・パターン抽出

### 琴平クイズ現状の評価

現在の琴平クイズは以下の構成:
- **スタート画面**: 難易度チップ + テーマチップ（ポップオーバーによるサブカテゴリ選択）
- **問題画面**: 縦並び4択ボタン + プログレスバー + カテゴリ/難易度バッジ
- **正解/不正解画面**: 絵文字(✅/❌) + 解説 + 出典リンク
- **結果画面**: スコア + 難易度別内訳 + ふりかえりアコーディオン
- **技術**: React + TailwindCSS、amber(#D4A574)をアクセントカラー

### ベストプラクティス抽出

#### 1. スタート画面: フィルター選択UIの最適解

| 調査から得た知見 | 琴平クイズの現状 | 改善案 |
|---|---|---|
| Kahootはミニマル1画面 | テーマチップ+ポップオーバーが複雑 | ポップオーバーを廃止し、全サブカテゴリをフラットに展開 |
| MD3フィルターチップは32dp高+48dpタッチ | チップの`py-2`(8px)は十分だがタッチ44pxに近い | `py-2.5`(10px)にしてタッチ領域を48px以上に確保 |
| Sporcleのカテゴリグリッドは直感的 | テーマ数が8で適切 | アイコン付きチップは現状で良い |
| 「すべて」がデフォルト選択は全サイト共通 | 実装済み | 維持 |

**具体的改善案:**
- ポップオーバー（ホバー/タップでサブカテゴリ表示）はモバイルで操作しにくい。サブカテゴリはデフォルトで非表示、テーマチップの長押しorタップでインライン展開に変更
- 選択中のフィルター条件を「バッジ」としてスタートボタン付近にサマリー表示
- 問題数0の組み合わせは事前にスタートボタンをdisabledにする（実装済み）

#### 2. 問題表示: 視認性・タップしやすさ

| 調査から得た知見 | 琴平クイズの現状 | 改善案 |
|---|---|---|
| Kahootの選択肢は色+図形で識別 | ABCDラベル+テキストのみ | 各選択肢に薄い背景色を追加（A=淡青, B=淡緑, C=淡黄, D=淡赤） |
| Apple HIG最小タップ44pt | `py-3`(12px)で高さ約48px、合格圏内 | 維持。ただし文字が多い選択肢は自動的に高さが増えるので問題なし |
| Quizletは1問に集中させるUI | 問題文+4択の縦並びは良い | 維持 |
| QuizKnockは問題文を大きく表示 | `text-lg`で適切 | 問題文を`text-xl`に拡大して視認性向上 |

**具体的改善案:**
- 選択肢ボタンにホバー時の微妙なスケールアニメーション（`hover:scale-[1.01]`）を追加
- 選択肢A-Dのラベルに背景色付き丸バッジを使い、視覚的な識別を容易にする
- 問題文のフォントサイズを`text-xl`に拡大

#### 3. 正解/不正解: フィードバックの演出

| 調査から得た知見 | 琴平クイズの現状 | 改善案 |
|---|---|---|
| Kahootは即座に色+音+アニメーション | ✅/❌の絵文字+bounce | 絵文字に加え、背景色の変化を追加 |
| 全サイトで緑=正解、赤=不正解が標準 | text-green-600/text-red-500で実装済み | 正解時にカード全体を薄緑背景に、不正解時に薄赤背景に |
| Quizletのスワイプアウトは気持ちいい | 静的な表示切替のみ | CSSトランジションでスライドインアニメーションを追加 |
| Kahootの速度ボーナスは盛り上がる | タイマーなし | タイマーは琴平クイズの「学習」性質に合わないため導入しない |

**具体的改善案:**
- 正解時: 画面上部に緑のフラッシュ + 紙吹雪の軽いCSSアニメーション（confetti）
- 不正解時: 画面が軽く揺れるシェイクアニメーション（0.3s）
- 「次の問題へ」ボタンにフォーカスを自動設定して、テンポよく進められるように
- 連続正解時に「3連続正解！」などのストリーク表示を追加

#### 4. 結果画面: リプレイ・共有の導線

| 調査から得た知見 | 琴平クイズの現状 | 改善案 |
|---|---|---|
| QuizKnockのSNSシェアが結果直下 | シェア機能なし | X(Twitter)シェアボタンを追加 |
| Sporcleのバッジ獲得通知 | 連続チャレンジ表示のみ | 称号システム（琴平博士/参拝名人等）を追加 |
| Kahootの表彰台アニメーション | 静的なスコア表示 | スコアの数字カウントアップアニメーション |
| Quizletの「間違えた問題だけ復習」 | アコーディオン式ふりかえりあり | 「間違えた問題だけもう一度」ボタンを追加 |
| けんてーごっこの合格/不合格判定 | パーセント表示のみ | 80%以上で「合格」バッジ表示 |

**具体的改善案:**
- SNSシェアボタン（X/Twitter）: 「琴平クイズで○/10問正解！ #琴平クイズ #金刀比羅宮」のテンプレートテキスト
- スコアの数字を0からカウントアップするアニメーション
- 正答率に応じた称号: 100%=「琴平博士」、80%+=「もの知り参拝者」、50%+=「見習い参拝者」
- 「間違えた問題だけ復習」ボタンを「次の10問へ」の下に追加

#### 5. 全体のトーン: 色・フォント・余白の方向性

| 要素 | 現状 | 改善案 |
|---|---|---|
| メインカラー | amber-600(#D97706) | 維持。金刀比羅宮の金色を想起させる良い選択 |
| 背景色 | bg-gray-50 | より温かみのあるbg-amber-50/bg-stone-50に変更 |
| フォント | システムデフォルト | Noto Sans JPを明示指定、見出しは700weight |
| 余白 | px-4 py-8（適切） | 維持 |
| アイコン | 絵文字ベース | 維持（SVGアイコンより絵文字の方が親しみやすい） |
| 角丸 | rounded-lg/rounded-xl | rounded-2xlに統一して柔らかい印象に |

**具体的改善案:**
- 背景をbg-gray-50からbg-stone-50に変更（温かみのあるトーン）
- ヘッダーにamber系のグラデーション背景を追加
- フォント: `font-family: 'Noto Sans JP', sans-serif`を明示
- 全体的にshadow-smを活用してカード要素に奥行きを追加

#### 6. モバイルファースト: タッチ操作の最適化

| 調査から得た知見 | 琴平クイズの現状 | 改善案 |
|---|---|---|
| Apple HIG 44pt最小タップ | 概ね達成 | ポップオーバー内の項目が小さい（py-2=8px） |
| MD3の48dpタッチターゲット | 選択肢ボタンはOK | テーマチップのタッチ領域を48px以上に統一 |
| Kahootの大きなボタン | スタートボタンpy-4で十分大きい | 維持 |
| スワイプジェスチャー | 未対応 | 結果画面の「次の問題へ」をスワイプでも遷移可能に（将来的） |

**具体的改善案:**
- ポップオーバーの各項目をpy-3(12px)に拡大
- テーマチップをmin-h-[44px]に設定
- 選択肢ボタン間のgap-3(12px)をgap-3.5(14px)に拡大して誤タップ防止
- スクロール時にヘッダーを固定（sticky header）してプログレスバーを常に表示
